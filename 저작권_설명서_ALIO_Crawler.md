# 프로그램 저작권 등록 설명서

## 1. 프로그램 개요

| 항목 | 내용 |
|------|------|
| 프로그램명 | ALIO Crawler (알리오 크롤러) |
| 버전 | 5.3 |
| 저작자 | 허재영 |
| 창작연도 | 2025년 |
| 최종 수정일 | 2026년 2월 20일 |
| 개발 언어 | Python 3 |
| 코드 규모 | 약 4,650줄 (단일 파일) |
| 실행 환경 | Windows / macOS (Python 3.10 이상) |
| 사용자 인터페이스 | GUI (tkinter) |

### 프로그램 목적 및 용도

ALIO(공공기관 경영정보 공개시스템, www.alio.go.kr)에 공시된 공공기관 경영정보를 체계적으로 수집하는 데이터 크롤링 도구이다. 공공기관의 징계처분 현황, 청렴도 평가, 내부규정, 안전사고 사망자수 등 11개 공시항목에 대해 기관유형, 주무부처, 지역별로 필터링하여 데이터를 수집하고, 엑셀 파일로 내보내거나 관련 첨부파일(PDF 등)을 일괄 다운로드할 수 있다.

---

## 2. 주요 기능

### 2.1 데이터 수집 (3단계 워크플로우)

1. **기관목록 수집**: ALIO API를 호출하여 공공기관 목록을 조회하고, 기관유형(공기업, 준정부기관 등), 주무부처(44개 부처), 지역(17개 시도)별로 필터링한다.
2. **공시내용 수집**: 선택한 기관의 공시 데이터를 수집하고, PDF 문서 및 첨부파일을 다운로드한다.
3. **엑셀 저장**: 수집된 데이터를 공시항목별 맞춤 컬럼 형식으로 엑셀(XLSX) 파일로 내보낸다.

### 2.2 지원 공시항목 (11종)

| 공시항목 | 설명 |
|---------|------|
| 내부규정 | 정관, 인사/복무/징계, 보수, 직제, 기타 규정 |
| 징계처분 현황 | 파면, 해임, 정직, 감봉, 견책, 기타 유형별 건수 |
| 청렴도 평가 결과 | 최근 5년간 연도별 청렴도 등급 |
| 사망자수 | 산업재해 및 안전사고 사망자수 (최근 5년) |
| 환경법규 위반현황 | 위반 건수, 제목, 일자 |
| 자체감사 결과 | 최근 5년 자체감사 결과 목록 및 첨부파일 |
| 경영실적 평가결과 | 기관별 경영실적 보고서 목록 및 첨부파일 |
| 일반현황 | 기관 설립근거 등 기본 정보 |
| 복리후생비 | 복리후생 관련 공시 데이터 |
| 수의계약 | 분기별 수의계약 현황 및 엑셀 첨부파일 |
| 징계제도 운영현황 | 징계제도 운영 관련 공시 데이터 |

### 2.3 GUI 기능

- 동적 컬럼 변경: 공시항목 선택에 따라 트리뷰 컬럼이 자동 변경
- 실시간 정렬: 컬럼 헤더 클릭 시 오름차순/내림차순 정렬
- 결과 내 검색: 수집된 데이터에서 키워드 필터링
- 진행률 표시: 진행 바 및 ETA(예상 소요시간) 계산
- 로그 관리: 실시간 로그 표시 (최대 10,000줄 제한으로 메모리 관리)
- 단축키: Ctrl+Q(종료), Ctrl+O(폴더 열기), Escape(중지)

### 2.4 데이터 처리

- 병렬 API 호출 (ThreadPoolExecutor, 최대 10~20개 워커)
- 요청 실패 시 지수 백오프 재시도
- 기관 중복 제거 (기관ID 기준 최신 공시 유지)
- HTML 파싱을 통한 상세 데이터 추출

## 3. 프로그램 구조

### 3.1 주요 클래스

| 클래스명 | 역할 |
|---------|------|
| ALIOCrawlerGUI | 메인 애플리케이션 (GUI 구성, API 연동, 파일 다운로드, 엑셀 저장) |
| BoundedText | 메모리 관리를 위한 줄 수 제한 텍스트 위젯 |
| ETACalculator | 작업 예상 소요시간 계산기 |

### 3.2 데이터 처리 흐름

```
사용자 입력 (기관유형/부처/지역/공시항목 선택)
    ↓
ALIO API 호출 (findOrganApbaList, itemOrganList 등)
    ↓
응답 데이터 파싱 및 필터링
    ↓
트리뷰에 결과 표시 (항목별 맞춤 컬럼)
    ↓
[선택] 공시 상세 수집 → PDF/첨부파일 다운로드
    ↓
[선택] 엑셀 파일로 내보내기 (항목별 맞춤 시트)
```

### 3.3 연동 API 목록

- `findOrganApbaList.json` - 기관 목록 조회
- `itemOrganListJung.json` - 정기 공시항목 조회
- `itemOrganListSusi.json` - 수시 공시항목 조회
- `itemReportListSusi.json` - 보고서 목록 조회
- `findRuleList.json` - 내부규정 목록 조회
- `findRuleDtl.json` - 규정 상세 조회
- `download/pdf.json`, `download/file.json` 등 - 파일 다운로드

## 4. 기술적 특징

### 4.1 독창적 구현 요소

- **11개 공시항목별 맞춤 파싱 로직**: 각 공시항목의 데이터 구조가 상이하므로, 항목별로 전용 파싱 및 표시 로직을 구현하였다.
- **동적 GUI 컬럼**: 선택한 공시항목에 따라 트리뷰의 컬럼 구성이 실시간으로 변경된다.
- **HTML 문서 파싱**: ALIO에서 제공하는 doc.html 형식의 공시 상세 페이지를 파싱하여 징계 유형별 건수, 위반 건수 등 구조화된 데이터를 추출한다.
- **엑셀 맞춤 내보내기**: 공시항목별로 서로 다른 컬럼 구조의 엑셀 시트를 생성한다.

### 4.2 사용 기술

- **GUI**: tkinter + ttk (순수 Python 표준 라이브러리)
- **HTTP 통신**: requests (세션 풀링, 타임아웃, SSL 인증서 처리)
- **HTML 파싱**: BeautifulSoup4
- **엑셀 생성**: openpyxl
- **병렬 처리**: concurrent.futures (ThreadPoolExecutor)
- **네트워킹**: 지수 백오프, 랜덤 지터, 요청 재시도
